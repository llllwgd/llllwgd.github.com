redis持久化

持久化的方式

  • rdb

  • aof

    rdm

    工作方式

    Redis 调用forks. 同时拥有父进程和子进程。
    子进程将数据集写入到一个临时 RDB 文件中。
    当子进程完成对新 RDB 文件的写入时，Redis 用新 RDB 文件替换原来的 RDB 文件，并删除旧的 RDB 文件。

    优点

  • 备份文件比较小

  • 支持多种备份模式同时存在， 比如每5分钟备份前30分钟，每1一天备份一个月的

  • 恢复速度比aof快

    缺点

  • 丢失最后一次备份之后的数据

  • 数据量大的时候备份影响系统性能

    aof

    Redis 调用forks. 同时拥有父进程和子进程。
    子进程将数据集写入到一个临时 RDB 文件中。
    当子进程完成对新 RDB 文件的写入时，Redis 用新 RDB 文件替换原来的 RDB 文件，并删除旧的 RDB 文件。

    优点

  • 故障时丢失的数据较少

  • 对文件进行重写

    缺点

  • 备份文件大

  • 恢复速度慢
